# Test 1: Say Name

The code below is there to demonstrate how automated tests work. 

At the top, you see the **application code**. Below, you see the automated test. 

> 💡 In a real-world application, the application code and test code would exist in two separate files. 

**Your task** is to read the code and try to understand it first. Then, click on "Run" and check the output. Try to understand what's going on and fix the code to get the test to pass.


---

When creating automated testing code, always remember these three simple steps: 

- 1️⃣ Arrange: Create **fake/sample data** to test your code with. 
- 2️⃣ Act: Execute the **application code** that's supposed to be tested.
- 3️⃣ Assert: Check if, given the **fake data** from before the **application code** does what you'd expect it to do.
